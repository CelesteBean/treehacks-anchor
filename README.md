## Overview

**Anchor** is an edge-deployed, real-time phone scam detection system designed to protect elderly and vulnerable individuals from phone-based fraud. The system captures audio from phone calls, transcribes speech using GPU-accelerated models, analyzes content for scam indicators, and generates spoken warnings when threats are detected.

### The Problem

Phone scams targeting elderly Americans cause billions of dollars in losses annually. Scammers use psychological manipulation tactics including:
- **Authority impersonation** â€” IRS, Social Security Administration, law enforcement
- **Fear and urgency** â€” Threats of arrest, account suspension, legal action  
- **Isolation** â€” "Don't tell your family about this call"
- **Financial pressure** â€” Gift cards, wire transfers, cryptocurrency

### The Solution

Anchor listens to phone conversations (from the user's side only), identifies scam indicators in real-time, and intervenes with spoken warnings before victims can be manipulated into sending money.

|---------|-------------|
| **Real-time Processing** | Sub-second latency from speech to detection |

## Installation

### Prerequisites

- **Hardware**: NVIDIA Jetson Orin Nano (8GB recommended)
- **OS**: JetPack 5.x (Ubuntu-based)
- **Python**: 3.10+
- **CUDA**: 11.4+

### Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/your-org/treehacks-anchor.git
cd treehacks-anchor

# 2. Create virtual environment
python3 -m venv venv
source venv/bin/activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. (Optional) Install LLM support for context-aware warnings
CMAKE_ARGS="-DLLAMA_CUDA=on" pip install llama-cpp-python

# 5. Download required models
# Whisper model: auto-downloaded by faster-whisper
# Piper TTS: Download from https://github.com/rhasspy/piper
mkdir -p models/piper
# Place en_US-lessac-medium.onnx in models/piper/

# (Optional) Qwen LLM for dynamic warnings
# Run: scripts/download_qwen_model.sh

# 6. Start the pipeline
./start_safe.sh
```

### Model Downloads

| Model | Size | Purpose | Location |
|-------|------|---------|----------|

| all-MiniLM-L6-v2 | ~80MB | Semantic similarity | Auto-downloaded |
| Piper TTS | ~50MB | Warning speech | `models/piper/` |
| Qwen2.5-0.5B | ~400MB | LLM warnings (optional) | `models/qwen-0.5b/` |

## Security Considerations

### Data Privacy

- **Local Processing**: All audio and analysis stays on-device
- **No Cloud Upload**: Zero data transmitted to external servers
- **Memory Cleanup**: Audio buffers cleared after processing

### Audio Handling

- Audio is captured but not stored persistently
- Base64 encoding for ZMQ transport only
- Logs do not include audio content

### Recommended Deployments

- Deploy on trusted hardware with physical security
- Limit network access to dashboard (LAN only recommended)
- Regularly update dependencies for security patches
